\section*{\hspace{12.5mm}Цель работы}
Цель работы: изучить и освоить разработку серверных веб-приложений на языке Java с использованием фреймворка Spring Boot, механизмов объектно-реляционного отображения (ORM) на примере Hibernate для работы с базой данных PostgreSQL, а также проектирование REST API для взаимодействия с графическим веб-интерфейсом на базе React.

\section*{\hspace{12.5mm}Задание}
На основании согласованной в первой и второй лабораторных работах предметной области разработать REST API серверное приложение на Java с использованием Spring Boot. Сведения об объектах предметной области необходимо хранить в базе данных PostgreSQL. Серверное приложение должно выступать полноценной заменой C\#-бэкенда и взаимодействовать с тем же графическим интерфейсом на React и базой данных.

\section*{\hspace{12.5mm}Решение}
Предметная область описывает управление гоночными командами трех различных серий (Formula 1, Formula 2, Formula E). Система позволяет хранить информацию о чемпионатах (сериях), гоночных командах и их пилотах, выполнять CRUD-операции, вычислять очки за позиции в гонке полиморфно по правилам конкретной серии, выполнять сравнение команд и предоставлять обобщенную статистику.

Программа содержит сущности JPA для отображения таблиц базы данных: <<RacingSeries>>, <<RacingTeam>>, <<RacingDriver>>. Для передачи данных и поддержки полиморфизма на уровне API разработаны объекты передачи данных (DTO): <<FormulaTeamDto>> и <<FormulaDriverDto>>.

Класс <<RacingTeam>> (сущность команды) содержит:
\begin{itemize}
  \item[--] Поля:
  \begin{itemize}
    \item[--] private Long id -- уникальный идентификатор (первичный ключ);
    \item[--] private String name -- название команды;
    \item[--] private String principalName -- имя руководителя;
    \item[--] private String baseLocation -- штаб-квартира;
    \item[--] private String teamColor -- фирменный цвет команды;
    \item[--] private Integer points -- набранные очки;
    \item[--] private Integer wins -- количество побед;
    \item[--] private Integer podiums -- количество подиумов;
    \item[--] private RacingSeries series -- связь со спортивной серией;
    \item[--] private List<RacingDriver> drivers -- список гонщиков команды;
    \item[--] Специфичные поля серий: F1 (\texttt{powerUnit}, \texttt{budgetCap}, \texttt{constructorPos}), F2 (\texttt{chassisModel}, \texttt{graduates}, \texttt{isFeeder}), FE (\texttt{energyPartner}, \texttt{batteryKwh}, \texttt{sustainScore});
  \end{itemize}
  \item[--] Методы: стандартные геттеры и сеттеры для всех полей.
\end{itemize}

Класс <<RacingDriver>> (сущность гонщика) содержит:
\begin{itemize}
  \item[--] Поля:
  \begin{itemize}
    \item[--] private Long id -- уникальный идентификатор;
    \item[--] private String firstName -- имя;
    \item[--] private String lastName -- фамилия;
    \item[--] private Integer racingNumber -- стартовый номер гонщика;
    \item[--] private Integer points -- набранные личные очки;
    \item[--] private RacingTeam team -- связь с командой;
  \end{itemize}
  \item[--] Методы: стандартные геттеры и сеттеры.
\end{itemize}

Класс <<RacingSeries>> (сущность серии) содержит:
\begin{itemize}
  \item[--] Поля:
  \begin{itemize}
    \item[--] private Long id -- уникальный идентификатор;
    \item[--] private String name -- название серии;
    \item[--] private String officialName -- официальное название;
    \item[--] private String headquarters -- штаб-квартира руководства;
    \item[--] private String governingBody -- руководящий орган;
    \item[--] private Integer foundedYear -- год основания;
    \item[--] private Integer rounds -- количество этапов в календаре;
    \item[--] private List<RacingTeam> teams -- список команд серии;
  \end{itemize}
  \item[--] Методы: стандартные геттеры и сеттеры.
\end{itemize}

Для связывания Java-классов с таблицами реляционной СУБД PostgreSQL используются аннотации JPA/Hibernate:
\begin{itemize}
  \item[--] @Entity -- объявляет класс сущностью базы данных;
  \item[--] @Table(name = "...") -- указывает имя соответствующей таблицы в СУБД;
  \item[--] @Id и @GeneratedValue(strategy = GenerationType.IDENTITY) -- определяют первичный ключ с автоинкрементом;
  \item[--] @Column(name = "...") -- задает имя колонки в таблице;
  \item[--] @ManyToOne и @JoinColumn(name = "...") -- настраивают связь <<многие к одному>> через внешний ключ (например, гонщик к команде);
  \item[--] @OneToMany(mappedBy = "...", cascade = CascadeType.ALL) -- настраивает обратную связь <<один ко многим>> с каскадным удалением зависимых записей.
\end{itemize}

Подключение к базе данных PostgreSQL настраивается в файле application.properties:
\begin{minted}{properties}
spring.datasource.url=jdbc:postgresql://localhost:5432/postgres
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.datasource.driver-class-name=org.postgresql.Driver
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=validate
\end{minted}

Для работы с сущностями в API используются репозитории Spring Data JPA, наследующие JpaRepository (например, RacingTeamRepository).

Для обработки HTTP-запросов и обмена данными с React-клиентом в JSON-формате разработан REST-контроллер <<RacingRestController>>. Он реализует следующие эндпоинты:
\begin{itemize}
  \item[--] GET /api/teams -- возвращает список Formula-команд (series\_id 1..3), отсортированных по очкам;
  \item[--] POST /api/teams -- создает новую команду с сохранением в БД;
  \item[--] PUT /api/teams/\{id\} -- обновляет данные команды;
  \item[--] DELETE /api/teams/\{id\} -- удаляет команду и каскадно связанных гонщиков;
  \item[--] GET /api/drivers -- возвращает список Formula-гонщиков;
  \item[--] POST /api/drivers -- регистрирует нового гонщика;
  \item[--] PUT /api/drivers/\{id\} -- обновляет информацию о гонщике;
  \item[--] DELETE /api/drivers/\{id\} -- удаляет гонщика из системы;
  \item[--] POST /api/teams/\{id\}/simulate-race -- симулирует гонку, полиморфно добавляя очки команде и выбранному пилоту по правилам серии (15-12-10... для F2, 25-18-15... для F1/FE);
  \item[--] GET /api/teams/compare -- сравнивает две команды по очкам (имитируя перегруженные операторы C\#);
  \item[--] POST /api/simulation/race -- глобальная симуляция гонки по переданному списку результатов;
  \item[--] GET /api/stats -- вычисляет общую статистику (лидеры, средние очки, количество участников) с фильтрацией по серии.
\end{itemize}

\section*{\hspace{12.5mm}Вывод}
В ходе выполнения лабораторной работы бэкенд гоночной системы управления был успешно перенесен с C\# ASP.NET Core на Java Spring Boot с использованием Hibernate JPA и PostgreSQL. Была реализована идентичная REST API маршрутизация и полиморфное поведение логики начисления очков, что позволило без каких-либо изменений на стороне клиента подключить разработанный React веб-интерфейс к новому Java-серверу. Были изучены принципы построения многослойной архитектуры в экосистеме Spring.
