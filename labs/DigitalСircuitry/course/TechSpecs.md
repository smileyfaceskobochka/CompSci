# Модульная оконная RGB‑матрица на WS2812B с I2C‑СК и МК на ESP32‑S3

## 1. Общая концепция

Система представляет собой **оконную RGB‑матрицу** из вертикальных полос на используя адресную светодиодную ленту WS2812B, каждая полоса управляется своим **СК (Слейв Контроллером)** на ATtiny85, а вся матрица - одним **МК (Мастер Контроллером)** на ESP32‑S3.

- Логическая матрица: **до 64 колонки × 128 пикселей** (8192 LED).
- Каждая колонка - **модуль‑полоса**: СК + до 128 светодиодов WS2812B.
- Связь МК ↔ СК: по I2C, **две независимые I2C‑шины** на ESP32‑S3.

***

## 2. Роли и термины

- **МК (Мастер Контроллер)**  
  - Платформа: **ESP32‑S3**.  
  - Функции:  
    - рендеринг эффектов для всей матрицы;  
    - управление двумя шинами I2C;  
    - логическая привязка колонок (СК) к координатам матрицы;  
    - пользовательский интерфейс (опционально: Сетевые интерфейсы управления).

- **СК (Слейв Контроллер)**  
  - Платформа: **ATtiny85**.  
  - Один СК = одна вертикальная полоса.  
  - Функции:  
    - приём команд по I2C;  
    - хранение буфера до 128 пикселей;  
    - генерация сигнала WS2812B для своей ленты;
    - локальная настройка собственного ID (номера колонки) через кнопки и отображение на ленте.

***

## 3. Аппаратная архитектура

### 3.1. МК на ESP32‑S3

Для избежания нестабильной работы I2C:

- Используются **две I2C‑шины**:
  - **I2C0 (шина A)** - обслуживает СК с колонками 0–31.
  - **I2C1 (шина B)** - обслуживает СК с колонками 32–63.  
- Каждая шина имеет выделенные пары GPIO (SDA_x/SCL_x).
- МК формирует единую картинку 64×128 и распределяет данные по колонкам и шинам.

### 3.2. СК на ATtiny85 (одна полоса)

- 5 В.  
- 8 МГц.  
- Распиновка (базовая):

  - PB5 — SCL (I2C).  
  - PB4 — SDA (I2C).  
  - PB3 — DATA на WS2812B (через резистор ~330 Ω).  
  - PB0 — кнопка `ID+`.  
  - PB1 — кнопка `ID−`.  
  - PB2 — резерв (опционально для диагностики).  

- Разъёмы:
  - **I2C/питание IN**: 5V, GND, SDA, SCL (от МК / предыдущего СК).  
  - **I2C/питание OUT**: 5V, GND, SDA, SCL (на следующий СК по цепочке).  
- Лента:
  - 5V/GND — от шины питания (толстый провод, с инжекцией по длине).  
  - DATA — от PB3 СК.

### 3.3. Лента

- Тип: WS2812BB или совместимая (5 В, 800 кГц).  
- На одну полосу/СК: ровно **128 пикселей**.  
- Инжекция питания рекомендована минимум в двух точках полосы.

***

## 4. Адресация и шины

### 4.1. Логический ID колонки

- Каждый СК хранит в EEPROM **логический номер колонки** `ID` в диапазоне **0–63**.  
- Этот номер:
  - используется МК для построения единой матрицы;  
  - отображается на ленте в режиме настройки.

### 4.2. Привязка ID к I2C‑адресу и шине

- **Шина A (I2C0)**:
  - обслуживает колонки с ID **0–31**.  
  - I2C‑адрес СК: `ADDR = 0x20 + ID` (0x20–0x3F).  

- **Шина B (I2C1)**:
  - обслуживает колонки с ID **32–63**.  
  - I2C‑адрес СК: `ADDR = 0x40 + (ID − 32)` (0x40–0x5F).

Таким образом:

- Максимум **64 уникальных СК** на одном ESP32‑S3 (по 32 на шину).  
- МК однозначно знает, на какой шине искать конкретный ID.

***

## 5. Режимы работы СК

### 5.1. Нормальный режим

- При старте:
  - СК читает `ID` из EEPROM.  
  - Вычисляет свой I2C‑адрес согласно диапазону (0x20–0x3F или 0x40–0x5F).  
- В нормальном режиме:
  - Работает как **I2C‑slave**, отвечает на запросы от МК.  
  - Хранит буфер 128×3 байта RGB.  
  - По командам МК обновляет буфер и выдаёт данные в ленту.

### 5.2. Режим настройки ID

Запуск:

- Вход в режим настройки при определённом условии (например, удержание кнопки при подаче питания, либо специальный жест).

Поведение:

- СК временно **не участвует в I2C‑обмене**.  
- Лента используется как **индикатор ID**:
  - на первых пикселях отображается `ID` (например, в двоичном виде по яркости/цвету);  
  - остальные пиксели гаснут или показывают специальный шаблон.  
- Кнопки:
  - Короткое нажатие `ID+`: увеличить ID (0–63, по кругу).  
  - Короткое нажатие `ID−`: уменьшить ID (0–63, по кругу).  
  - Длительное удержание `ID+` (например, ≥ 3 с):  
    - сохранить новый ID в EEPROM;  
    - выйти из режима настройки;  
    - перейти в нормальный режим с новым ID → новым I2C‑адресом.

Пользователь настраивает порядок колонок, глядя на саму полосу (никаких джамперов, дисплеев и программаторов).

***

## 6. Протокол обмена МК ↔ СК (I2C)

Пример пакета от МК к СК:

- `[CMD : 1 байт][LEN : 1 байт][PAYLOAD : LEN байт][опциональный CRC]`.

Минимальный набор команд:

1. **FILL (0x01)**  
   - Заполнить всю полосу одним цветом.  
   - `LEN = 3`, `PAYLOAD = [R][G][B]`.

2. **PIXEL (0x02)**  
   - Установить цвет одного пикселя.  
   - `LEN = 4`, `PAYLOAD = [pos][R][G][B]`.

3. **SHOW (0x03)**  
   - Отобразить текущий буфер на ленте.  
   - `LEN = 0`.

4. **EFFECT (0x10)** (опционально)  
   - Включить локальный эффект на СК, чтобы не гонять полный буфер по шине.  
   - `LEN ≥ 1`, `PAYLOAD = [effect_id][параметры...]`.

5. **PING/STATUS (0xFF)** (опционально)  
   - Проверка связи и получение простого статуса.

Подход к синхронизации:

- МК может либо вызывать `SHOW` по всем СК после обновления данных, либо СК сам обновляет ленту, как только буфер изменён и пришёл полный набор данных.

***

## 7. МК (ESP32‑S3): логика

Задачи МК:

- Инициализировать две I2C‑шины (I2C0 и I2C1) на выбранных GPIO.  
- При старте:
  - просканировать адресные диапазоны 0x20–0x3F и 0x40–0x5F на соответствующих шинах;  
  - составить карту: какие ID/колонки реально присутствуют.
- Внутренне представлять **единую матрицу до 64×128**:
  - ось X — колонка (ID 0–63),  
  - ось Y — пиксель 0–127 вдоль полосы.  
- Для каждого кадра:
  - рендерить картинку/эффект в общий буфер;  
  - разбивать по колонкам;  
  - отправлять нужные команды (FILL/PIXEL/EFFECT/SHOW) по соответствующей шине и адресу.

***

## 8. Питание (кратко)

- Каждый СК (ATtiny85): порядка **5–10 мА**, суммарно при 64 СК < 0.7 А — можно считать пренебрежимым относительно лент.  
- Одна полоса (128 WS2812B):
  - максимальный теоретический ток до ~7.7 А (полный белый 255,255,255);  
  - в реальных эффектах при разумной яркости — обычно 2–3 А.  
- 64 полосы:
  - типичный суммарный ток лент: 64 × (2–3 А) = 128–192 А,  
  - это уже уровень тяжёлого стационарного инсталляционного БП и толстой разводки — в практическом варианте, скорее всего, яркость будет ограничена, а система масштабирована по блокам.

В рамках ТЗ важно:

- закладывать **отдельную силовую разводку для лент** (с инжекцией),  
- шины I2C + питание СК держать отдельно и с малым током.

***

## 9. Модульность

- Все СК одинаковые по железу и прошивке.  
- Логический порядок колонок определяется **только ID на каждом СК**, который пользователь сам выбирает через кнопки, глядя на отображение ID на ленте.  
- МК воспринимает матрицу как набор колонок с ID 0–63, независимо от физического расположения и шины (A/B) — это задаётся в коде.
