\section*{Цели лабораторной работы}
\begin{itemize}
  \item[$-$] изучить принципы инкапсуляции, наследования и полиморфизма в языке C++;
  \item[$-$] научиться проектировать иерархию классов с использованием абстрактных базовых классов и виртуальных функций;
  \item[$-$] освоить работу с объектами через указатели базового типа и использование векторов смарт-указателей.
\end{itemize}

\section*{Задание}
Разработать приложение <<Formula Racing Management System>>, демонстрирующее принципы ООП, со следующими требованиями:
\begin{enumerate}
  \item Создать иерархию классов: один родительский (абстрактный) и три дочерних (F1Team, F2Team, FormulaETeam).
  \item В каждом классе определить не менее двух данных-членов и двух собственных методов.
  \item Для дочерних классов реализовать не менее двух унаследованных и двух перекрытых функций.
  \item Доступ к полям реализовать только через геттеры и сеттеры.
  \item Обеспечить наличие не менее 3-х конструкторов в дочерних классах.
  \item Работу с объектами осуществлять только через переменные родительского типа (полиморфизм).
  \item Каждый класс должен находиться в отдельном файле.
  \item Продемонстрировать перегрузку операторов ($+=, ==, >, <<$).
\end{enumerate}

\section*{Реализация приложения}

\subsection*{Предметная область}
Система предназначена для управления гоночными командами различных серий (Formula 1, Formula 2 и Formula E). Каждая серия имеет свои особенности начисления очков, бюджетные ограничения и технические параметры, что делает их идеальными кандидатами для демонстрации наследования и полиморфизма.

Описание иерархии:
\begin{itemize}
  \item[$-$] \texttt{FormulaTeam} (Abstract) $-$ базовый класс, хранящий общую информацию (название, руководитель, очки, победы, подиумы). Содержит чистые виртуальные методы displayInfo(), getSeriesName(), calculatePoints();
  \item[$-$] \texttt{F1Team} $-$ дочерний класс, добавляющий параметры бюджета, позиции в чемпионате и силовой установки.
  \item[$-$] \texttt{F2Team} $-$ дочерний класс, фокусирующийся на подготовке пилотов (выпускники F1) и модели шасси.
  \item[$-$] \texttt{FormulaETeam} $-$ дочерний класс, учитывающий экологический рейтинг, емкость батарей и энергетических партнеров.
  \item[$-$] \texttt{FormulaDriver} $-$ вспомогательный класс (агрегируется в \texttt{FormulaTeam}), хранящий информацию о гонщике (имя, фамилия, номер, набранные очки).
\end{itemize}

\subsection*{Особенности реализации}

\subsubsection*{Абстрактный базовый класс}
Базовый класс \texttt{FormulaTeam} является абстрактным, так как содержит чистые виртуальные функции. Это гарантирует, что мы не сможем создать экземпляр общего типа <<Команда>>, а будем работать только с конкретными реализациями.

\begin{minted}{cpp}
virtual void displayInfo() const = 0;
virtual std::string getSeriesName() const = 0;
virtual int calculatePoints(int position) const = 0;
\end{minted}

\subsubsection*{Полиморфизм, векторы и смарт-указатели}
Для автоматического управления временем жизни объектов в куче используются умные указатели \texttt{std::shared\_ptr} и \texttt{std::unique\_ptr}. В системе используется контейнер \texttt{std::vector} для хранения полиморфных указателей на базовый класс \texttt{FormulaTeam}. Объекты дочерних классов создаются динамически и помещаются в этот вектор (upcasting), что позволяет хранить и обрабатывать их единообразно:

\begin{minted}{cpp}
std::vector<std::unique_ptr<FormulaTeam>> poly_teams;

// Добавление объектов дочерних классов в вектор
poly_teams.push_back(std::make_unique<F1Team>("McLaren", "Stella", "UK", "Mercedes", 135.0, 2, 291, 5, 12));
poly_teams.push_back(std::make_unique<F2Team>("Prema", "Rosin", "Italy", "Dallara F2", 0, true, 89, 2, 5));

// Чтение из вектора и вызов виртуальных методов (полиморфизм)
for (size_t i = 0; i < poly_teams.size(); ++i) {
    std::string series = poly_teams[i]->getSeriesName(); // Вызов переопределенного метода
    std::string summary = poly_teams[i]->getSummary();
}
\end{minted}

\subsubsection*{Приведение типов (static\_cast и dynamic\_cast)}
Для вызова специфичных методов дочерних классов, которые отсутствуют в базовом классе \texttt{FormulaTeam}, применяются операции нисходящего приведения типов (downcasting). 

В приложении используется \texttt{dynamic\_cast} для безопасного приведения типов во время выполнения. Он использует механизм RTTI (Run-Time Type Information) и возвращает \texttt{nullptr}, если преобразование невозможно:

\begin{minted}{cpp}
if (auto *f1 = dynamic_cast<F1Team *>(poly_teams[0].get())) {
    // Успешно: объект действительно является F1Team
    std::string puStatus = f1->getPowerUnitStatus();
} else {
    // Ошибка приведения типов, объект относится к другой серии
}
\end{minted}

Также для случаев, когда тип объекта точно известен программе на этапе компиляции, применяется быстрое статическое приведение типов без дополнительных проверок с помощью \texttt{static\_cast}:

\begin{minted}{cpp}
if (poly_teams[0]->getSeriesName() == "Formula 1") {
    F1Team *f1 = static_cast<F1Team *>(poly_teams[0].get());
    std::string puStatus = f1->getPowerUnitStatus();
}
\end{minted}


\subsubsection*{Перегрузка операторов}
Реализована перегрузка операторов для удобной работы с объектами:
\begin{itemize}
  \item \texttt{operator+=} $-$ добавление очков;
  \item \texttt{operator==} $-$ сравнение команд по названию;
  \item \texttt{operator\textgreater} $-$ сравнение по количеству очков (используется для сортировки);
  \item \texttt{operator\textless\textless} $-$ потоковый вывод краткой информации.
\end{itemize}
